hexo常用命令

初始化hexo

$ hexo init dirname
$ npm install

创建新发布

$ hexo new "My New Post"
More info: Writing Writing-Chinese

启动server

$ hexo server

或者

$ hexo s
More info: Server

生成静态文件

$ hexo generate

或者

$ hexo g
More info: Generating

部署到远程地址

$ hexo deploy

或者

$ hexo d

更换新的主题
克隆新的主题到theme目录

$ git clone https://github.com/litten/hexo-theme-yilia.git themes/yilia

打开_config.yaml把theme字段的值修改为刚才在theme目录里的主题名

theme: yilia

clear完以后重新部署

$ hexo g
$ hexo s
$ hexo d

解决信息:

hexo init
npm install
hexo支持mathjax
安装

$ npm install hexo-math --save
在hexo目录初始化

$ hexo math install
配置

plugins:
- hexo-math

在blog当前目录里执行下面命令

$ hexo math install

在_config.yml文件中添加：

plugins: hexo-math

hexo支持category与tag
打开scaffolds/post.md, 修改内容为

---
title: {{ title }}
date: {{ date }}
tags:
categories:
---

以后执行hexo n ‘name’生成的md文件中就会有category了

hexo支持版权声明
安装

npm install hexo-filter-indicate-the-source --save

配置

indicate_the_source:
  pattern: indicate-the-source
  enable: true
  render_engine: ejs
  element_class: source
  domain_white_list:
     - localhost
  template: "<blockquote><p>转载请注明出处：<%- post.permalink %></p><p>访问原文「<a href='<%- post.permalink %>'><%- post.title %></a>」</p></blockquote>"

使用的时候在md文中合适的位置插入

<!-- indicate-the-source -->

git常用命令
设置代理

git config --global https.proxy http://127.0.0.1:1080
git config --global https.proxy https://127.0.0.1:1080
git config --global --unset http.proxy
git config --global --unset https.proxy

修改远程分支命令

git remote set-url origin [url]

删除分支与添加remote分支

git remote rm origin
git remote add origin [url]

用远程的覆盖本地

git fetch --all  
git reset --hard origin/master

设置upstream

git branch --set-upstream-to=origin/dev
或者
git branch -u origin/dev

取消当前分支的upstream

git branch --unset-upstream

取消指定分支的upstream

git branch --unset-upstream [分支名]

查看upstream，打开.git/config文件
报错Git: fatal: Pathspec is in submodule

git rm --cached directory
git add directory
